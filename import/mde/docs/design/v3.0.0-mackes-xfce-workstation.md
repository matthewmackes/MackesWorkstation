# Mackes XFCE Workstation 1.0.0 — Design Lock

**Locked:** 2026-05-18
**Status:** Design-only — no implementation work has begun.
**Source:** 50-question survey (mackes-worklist-management skill, survey-style workflow).
**Estimated effort:** 5–6 months sustained.
**Goal (2026-05-18):** Complete + release full 1.0. Outstanding design. Desktop must be very, very fast with a low memory footprint — Q41 revised accordingly (see §13).

This document captures the decisions made in the 50-question design survey
that locks the shape of the v3.0.0 / 1.0.0 rewrite. It supersedes the
prior v2.x trajectory in `docs/MACKES_SHELL_SPEC.md` for forward work.

## 4. Form factor

Q7 lock: **Two stripes — top status bar + bottom dock (macOS-class layout).**

### Top bar

| Trait | Value |
|---|---|
| Height | 20 px |
| Icons | Mackes-Carbon monochrome only |
| Background | Opaque PatternFly dark `#151515` + 1 px hairline border |
| Multi-monitor | Renders on every monitor (Q17) |
| Contents (Q8) | Left: Apple-menu button + global appmenu (DBusMenu) · Center: clock + calendar dropdown · Right: status cluster |
| Status-cluster click (Q28) | Slides in the v2.2.0 Notification Drawer from the right |

### Bottom dock

| Trait | Value |
|---|---|
| Icon size | 48 px (Q12) |
| Magnification | None — predictable, no hover scaling |
| Auto-hide | None — always visible, strut-reserved (Q11) |
| Icon style | Monochrome Mackes-Carbon symbolic, every item (Q14) |
| Background | Same PatternFly dark + hairline border (Q15) |
| Multi-monitor | Primary monitor only (Q17) |
| Items (Q9) | Pinned launchers + running indicators + discovered mesh resources, **interleaved** (Q10) |

### State indicators on dock icons (Q16)

Two channels per icon:
- 1 px dot under icon: `muted` = running · `accent` = focused · `alert` = urgent
- Right-edge superscript badge: numeric for unread, glyph for urgent

## 6. Window management (Q30, Q31, Q35)

| Action | Hotkey |
|---|---|
| App switcher (horizontal previews) | `Super+Tab` |
| Exposé grid (all windows tiled) | `F3` or top-right hot corner |
| Apple menu / Spotlight | `Super+Space` |
| Quit active app | `Super+Q` |
| Close active window | `Super+W` |
| Lock screen | `Super+L` |
| Clipboard history | `Super+V` |
| Open Thunar at `~/QNM-Shared/` | `Super+E` |

**Cmd key = Super** (the Windows / Penguin key).

Hotkey policy (Q35): the panel claims **all 8 bindings** at install. Any
existing user binding on those keys is preserved as a one-time backup.

## 7. Iconography (Q13, Q14, Q32)

- **Source of truth:** Mackes-Carbon (shipped in 2.3.0, derived from IBM
  Carbon Design System, Apache 2.0). All panel icons resolve from this
  theme.
- **Monochrome everywhere** — top bar AND dock. Apps lose their brand
  color to gain visual consistency. Implication: we need an app →
  Carbon-icon lookup table (similar pattern to the freedesktop name map
  shipped in `install-helpers/mackes-carbon.map`); fallback is a generic
  Carbon `application` glyph.
- **Nerd Font glyphs** (Q32) appear only in inline-text contexts: Apple
  menu status-line items (e.g. `󰷨 Mesh: 7 peers`), small inline
  battery/wifi/cpu indicators where a Carbon SVG would lose detail at
  size. Carbon SVG is the source for every other context.

## 9. Thunar integration (Q27, Q38)

- Default Thunar start folder = `~/QNM-Shared/` (xfconf edit applied
  during birthright)
- `Super+E` opens Thunar at this default location
- Dock drag-and-drop: **pin/reorder only** (Q38) — no file-drop-onto-
  app-icon, no mesh-send-via-drop. Mesh sends happen through the peer
  action popover (see §11).

---

## 10. Mesh integration (Q9, Q10, Q33, Q34)

Mesh peers and discovered mesh resources (peers, mesh-mounted folders,
mesh-advertised services) appear as **first-class dock items**,
interleaved with apps.

- Mesh peer click (Q34) → popover with action menu: Files (Thunar at
  `~/QNM-Shared/peer-name/`) · SSH · RDP · VNC · Services · Send file…
- No separate top-bar mesh glyph (Q33) — the Notification Drawer's Mesh
  section is the secondary surface.

---

## 11. LightDM greeter (Q36)

The greeter renders a fake top bar matching the running panel: 20 px
dark stripe, Carbon glyphs, Red Hat fonts. Visual continuity from boot
to first paint of the desktop. Implemented in
`lightdm-gtk-greeter.conf` + a small CSS overlay we ship.

---

## 12. Plymouth boot splash (Q37)

Rebuilt theme:
- Black background
- Centered Mackes logo (existing branding)
- 20 px thin progress line at the bottom of the screen, matching the
  dock's vertical position
- Continuity Plymouth → LightDM → mackes-panel
