## 1. Headline

The Mackes mesh is one user's many machines, not N independent
desktops. Today every peer fires every cross-peer notification
(3 peers × one brick-down event = 3 toasts), DND state is per-peer
despite v6.0's mesh-wide design intent, and the MON-3/4 alert
pipeline doesn't actually mesh-replicate despite its preamble
(`mde-alert-emit` writes to `$XDG_DATA_HOME` local, not to the
mesh). v5.1 fixes all three with one composed redesign.

Three moves compose:

1. **Attended-peer routing.** Every peer publishes idle + lock
   state; the lowest-idle unlocked peer is "attended"; the router
   fires toasts there. Roaming: wake another peer within 15 min
   and unseen events follow with a `[Late]` decoration.
2. **Unified mesh notification bus.** Every notification (FDO,
   Netdata-alert, KDC2-bridge) lands in the same JSON schema at
   `<qnm_root>/<self>/mackesd/notifications/<ulid>.json`. One
   `alert_router` worker reads every peer's dir, applies fanout
   policy, fires `notify-send` where it should fire, writes to
   the local DB always so history-panel is the union.
3. **Focus modes.** Three named filters (`work` / `quiet` / `off`)
   per peer. Birthright sets lighthouses to `off` and every other
   peer to `work` by default. Router consults active mode's
   `allow_classes` / `block_classes` globs before firing.

## 3. Architecture

### 3.1 Unified notification schema (Move 2)

Every notification — whether emitted by `mde-alert-emit` (Netdata),
`mded::Notify` (FDO clients), or `notification_relay` (KDC2 phone
bridge) — writes to
`<qnm_root>/<self>/mackesd/notifications/<ulid>.json`:

```json
{
  "id": "01HXXX...",
  "schema_version": 1,
  "class": "gluster.conflict",
  "scope": "fleet",
  "fanout": "attended",
  "urgency": "critical",
  "origin_peer_id": "laptop-02",
  "origin_unix_s": 1716700100,
  "title": "Split-brain on ~/Documents/notes.md",
  "body": "Peers desk-01 + laptop-02 wrote concurrent versions...",
  "thread_id": "gfid:7a8b9c...",
  "actions": [
    {"id": "keep-mine",   "label": "Keep mine"},
    {"id": "keep-theirs", "label": "Keep theirs"},
    {"id": "open-diff",   "label": "Open diff…"}
  ],
  "chart_url": "https://lighthouse:19999/...",
  "ttl_unix_s": 1716800000,
  "dismissed_at": null
}
```

`scope` ∈ `{fleet, peer, user, security}` (informational; the
fanout is what actually routes).
`fanout` ∈ `{attended, origin, aggregator, all}`.
`urgency` ∈ `{low, normal, critical}` → maps to `notify-send`
`--urgency`.

### 3.2 Class fanout table (Move 1 + Move 2)

Scope-driven defaults per Q1 lock. Class glob patterns; first
match wins; unmatched class defaults to `attended`.

| Class glob | Scope | Default fanout |
|---|---|---|
| `gluster.quorum`, `gluster.brick`, `gluster.quota` | fleet | Aggregator |
| `gluster.conflict`, `gluster.heal` | fleet | Attended |
| `gluster.peer.*` (added/removed) | fleet | Aggregator |
| `nebula.lighthouse`, `nebula.handshake` | fleet | Aggregator |
| `workstation.thermal`, `workstation.disk`, `workstation.dnf-pending` | peer | Origin |
| `kdc2.notif.*`, `kdc2.call.*` | user | Attended |
| `voice.call`, `voice.missed`, `voice.voicemail` | user | Attended |
| `security.*`, `audit.*` | security | All |
| `presenter.*` | user | Attended |
| (unmatched) | (default) | Attended |

Pure-fn `default_fanout_for_class(class: &str) -> Fanout` exported
+ unit-tested. Emitters may override per-event via the JSON
`fanout` field.

### 3.3 Attendance election (Move 1)

Each peer's mded publishes
`<qnm_root>/<self>/mackesd/attendance.json` every 30 s:

```json
{
  "node_id": "desk-01",
  "last_input_unix_s": 1716700000,
  "lock_state": "unlocked",
  "schema_version": 1
}
```

`last_input_unix_s` sourced via `org.freedesktop.IdleMonitor`
(sway exposes it, already a dep for mde-portal screensaver
gating). `lock_state` from `org.freedesktop.ScreenSaver` on
sway-side (`is_locked()` proxy).

Pure-fn `elect_attended(snapshots, now_unix_s) -> Option<NodeId>`:

* Drop snapshots older than 5 min (multiple of 30 s publish
  cadence) — covers stalled publisher.
* Filter to peers with `lock_state == "unlocked"`.
* If empty, return `None` (no attended peer; alerts queue for
  the 15-min roaming grace per §3.4).
* Otherwise pick the peer with the lowest
  `now_unix_s - last_input_unix_s`. Ties broken by node_id
  lexicographic.

### 3.4 Roaming grace (Move 1, Q2 lock = 15 min)

When `alert_router` on a peer detects this peer just became
attended (was not the elected peer in the previous tick, is now),
it scans events from `now - 900 s` to `now` that:

* `dismissed_at == null`
* `fanout == "Attended"` (other fanouts already fired on their
  intended peer)
* This peer hasn't already fired `notify-send` for the event
  (tracked in local SQLite `toasted_locally` table keyed on
  event id)

For each match, fire `notify-send` with `[Late]` prefix in the
title. Mark `toasted_locally` row inserted.

### 3.5 Focus modes (Move 3, Q3 + Q4 locks)

Each peer has `<qnm_root>/<self>/mackesd/focus-profile.json`:

```json
{
  "active_mode": "work",
  "schema_version": 1,
  "modes": {
    "work":  {
      "allow_classes":  ["*"],
      "block_classes":  [],
      "pierce_dnd":     ["*.critical"]
    },
    "quiet": {
      "allow_classes":  ["*.critical", "voice.call", "security.*"],
      "block_classes":  ["*"],
      "pierce_dnd":     ["voice.call", "security.*"]
    },
    "off": {
      "allow_classes":  [],
      "block_classes":  ["*"],
      "pierce_dnd":     []
    }
  }
}
```

Glob match semantics: `*` matches one or more classes at one
level (`gluster.*` matches `gluster.conflict` AND `gluster.heal`
but NOT `gluster.peer.added`). Bare `*` matches anything. `*.foo`
matches anything ending in `.foo` at the leaf level.

A class is allowed iff:
`any(allow_match(class)) && !any(block_match(class))`.

DND pierce: when the operator has Portal::ToggleDND on, only
events whose class matches an entry in the active mode's
`pierce_dnd` list still fire. All other notifications go to DB
history but skip the toast.

Pure-fn `matches_class(pattern: &str, class: &str) -> bool` +
`is_allowed(profile_mode, class, dnd_on) -> Decision` exported
+ unit-tested per (class, mode, dnd_on) row.

Birthright role-default per Q4:
* `role: lighthouse` → `active_mode: off`
* every other role → `active_mode: work`

Operator overrides via Portal status-zone focus glyph (click
cycles work → quiet → off → work) or Workbench Notifications
focus-mode editor panel.

### 3.6 Router pipeline

```
notification.json appears (any peer's QNM-Shared dir)
    │
    ▼
alert_router on every peer reads it within 2s
    │
    ├──► local NotificationsService DB write
    │    (always — history panel sees everything)
    │
    ▼
event.fanout decides candidate peers (Move 1):
    Attended   → elect_attended() == self → fire
    Origin     → event.origin_peer_id == self → fire
    Aggregator → netdata_aggregator leader == self → fire
    All        → fire on every peer
    │
    ▼
focus_profile.active_mode filter (Move 3):
    is_allowed(active_mode, class, dnd_on) == Allowed → continue
    otherwise → no toast (DB write happened above)
    │
    ▼
notify-send with actions from event.actions
mark toasted_locally row inserted
```

### 3.7 Coalescing + TTL pruning

Closes GF-16.3 via data-shape instead of class hardcoding.

`thread_id` lets router collapse all events sharing the same id
into one toast that grows in place (FDO `replaces_id`). Threshold:
≥3 events in the same thread within 10 s. Coalesced toast title:
`"<count> events in <thread>"`.

`ttl_unix_s` lets every peer's tick `unlink()` events past their
TTL without a central garbage collector. Default TTL:
* `critical` → 7 d
* `normal` → 24 h
* `low` → 1 h

Pure-fn `should_prune(event, now) -> bool` exported + tested.

## 4. Hard-cut migration (Q5 lock)

* `crates/mde-alert-emit/src/main.rs::default_output_dir()` changes
  from `$XDG_DATA_HOME/mde/alerts/` to
  `<qnm_root>/<self>/mackesd/notifications/`.
* `crates/mackesd/src/workers/alert_relay.rs` renames to
  `alert_router.rs`; default scan dir changes from
  `~/.local/share/mde/alerts/` to
  `<qnm_root>/*/mackesd/notifications/` (every peer's dir).
* v5.1 CHANGELOG documents the breaking change explicitly:
  "any external consumer of `~/.local/share/mde/alerts/` must
  update to `<qnm_root>/<peer>/mackesd/notifications/`. No
  backward-compat shim — alert events now mesh-replicate
  across peers via QNM-Shared."
* No `--legacy-output-dir` flag, no compat symlink, no
  read-both. Per §0.12 and Q5 lock.

## 6. Acceptance criteria

The v5.1 cut is greenlit when each of these passes on a 3-peer
bench:

1. **Schema + migration.** `mde-alert-emit --class=test.alert
   --dry-run-from-env` writes to
   `<qnm_root>/<self>/mackesd/notifications/<ulid>.json`. `ls
   ~/.local/share/mde/alerts/` returns ENOENT (or operator has
   stale legacy data; new emits no longer land there).
2. **Attendance election.** Three peers all unlocked, user types
   on peer-A → within 60 s `attendance.json` shows peer-A lowest
   idle → trigger a `class: voice.call` event → only peer-A fires
   `notify-send`.
3. **Roaming grace.** From state above, lock peer-A, leave fleet
   idle for 5 min, then type on peer-B → within 60 s
   `attendance.json` shows peer-B attended → within next tick
   peer-B fires unseen Attended-class events from last 15 min
   with `[Late]` prefix.
4. **Class fanout.** Trigger `gluster_quorum_lost` → only Netdata
   aggregator (one peer) fires toast (was all three).
   Trigger `workstation.thermal` on peer-A → only peer-A fires.
   Trigger `security.audit` → all three fire.
5. **Focus modes.** Click Portal status-zone focus glyph on
   peer-B → cycles work → quiet → off → work. With peer-B in
   `off`, trigger `voice.call` → peer-B silent; DB row written;
   history panel shows the entry.
6. **DND pierce.** With Portal::ToggleDND on + active_mode = work,
   trigger `gluster.quorum.critical` → fires (pierce_dnd matched).
   Trigger `gluster.heal` → doesn't fire.
7. **Coalescing.** Trigger 50 events with same `thread_id` → 1
   toast with count = 50 + grows in place via replaces_id.
8. **TTL pruning.** Drop an event with `ttl_unix_s = now - 1` →
   next router tick `unlink()`s it.
9. **Birthright role-default.** `apply_focus_default(role="host")`
   writes `focus-profile.json` with `active_mode: work`. Same
   with `role="lighthouse"` → `active_mode: off`.
10. **No silent broken consumers.** Any external script that
    referenced `~/.local/share/mde/alerts/` returns ENOENT —
    operator-visible failure (not silent drift).

## 7. Out-of-scope (deferred)

* **Cross-mesh notification federation.** Two-mesh-bridge users
  (rare) keep two separate notification streams in v5.1.
* **Per-app notification rules.** "Mute Slack toasts on this peer"
  beyond class-glob granularity. Deferred to v5.2.
* **Notification-driven actions on remote peers.** Click "Keep
  mine" on peer-A for a conflict that originated on peer-B —
  the action's `Gluster.Status::ResolveConflict` D-Bus call has
  to bridge peers. v5.2 (currently single-peer only).
* **Per-class custom timeouts.** All notifications use
  `notify-send` defaults; per-class display-duration is v5.2.

## 8. Risks

* **Attendance file staleness.** A peer's publisher tick stalls
  (crash, FUSE hang) → attendance.json gets stale. Mitigation:
  `elect_attended` drops snapshots older than 5 min (multiple of
  30 s publish cadence) — at worst a 5-min routing delay.
* **Two-attended ties.** Two peers both unlocked + identical
  idle → tie-break on node_id lex is deterministic but might
  surprise the operator. Mitigation: status glyph shows which
  peer is attended-by-election; click on glyph reveals tooltip
  "attended via lowest-idle-tie-break vs peer-X."
* **Hard-cut breaks unknown consumers.** Operators with custom
  shell scripts reading `~/.local/share/mde/alerts/` get
  ENOENT. Mitigation: CHANGELOG callout + v5.1 release notes;
  no backward-compat shim per Q5 lock.
* **`thread_id` collisions across event classes.** Two unrelated
  events accidentally share a thread_id → coalesced incorrectly.
  Mitigation: emitter convention is `<class>:<unique>` (e.g.
  `gluster.conflict:gfid:7a8b9c`), enforced by lint.
* **Focus-mode profile JSON drift.** Operator hand-edits
  `focus-profile.json` and corrupts the schema → router falls
  back to `mode: work` with a warning toast. Validation is
  best-effort, not blocking.
