# v5.0.0 — Auto CUPS print sharing + sync (`cups_sync`)

**Status:** Locked 2026-05-29 via 8-Q survey (round 1 of a 15-Q
print + firewall-monitor session).
**Target release:** v5.0.0 — **operator-elevated to cut-blocking core**
(§11.1) 2026-05-29, overriding the §0.17 minimal-core default.
**Depends on:** Nebula fabric + `mesh-storage` (LizardFS) mount +
Bus foundation live on every target peer.
**Worklist:** `docs/PROJECT_WORKLIST.md` § "PRINT-1..PRINT-10".

## 1. Headline

A printer configured on any MDE peer auto-appears — and is actually
usable — on every other peer. The host peer's CUPS is the print server
for its local printers; other peers import a remote queue that routes
jobs over the Nebula overlay to the host, which spools to the hardware.
No multicast, no operator step: a `mackesd::cups_sync` worker converges
printer definitions (and PPDs, defaults, presets) through the
`mesh-storage` volume the same write-own-file / read-union way the
PEERVER peer-data converges.

## 3. Architecture

### 3.1 Convergence (the file layer)

```
peer A (cups_sync, 5s tick)                 mesh-storage (LizardFS)
  lpstat -e / -p / -l  ──────────────►  <mesh-storage>/printers/anvil.json
  cupsctl + per-queue PPD ────────────►  <mesh-storage>/printers/ppd/anvil/<queue>.ppd
                                            <mesh-storage>/printers/_defaults.json   (fleet default + presets, LWW)
peer B (cups_sync, 5s tick)
  read union of printers/*.json  ─────►  lpadmin -p "<queue>@anvil" \
                                            -E -v ipp://10.42.0.x:631/printers/<queue> \
                                            -m everywhere   (or -P <replicated PPD> for legacy)
  prune local "<q>@<host>" queues whose host-file vanished
```

* **Own file:** `<mesh-storage>/printers/<host>.json` =
  `{host, overlay_ip, queues: [{name, ipp_path, kind: "everywhere"|"ppd",
  ppd_ref?}], written_at_ms}`. Authority over its own row, exactly like
  PEERVER.
* **PPD store:** legacy-printer PPDs replicate under
  `<mesh-storage>/printers/ppd/<host>/<queue>.ppd` (Q4). IPP-Everywhere
  queues carry no PPD.
* **Defaults + presets:** `<mesh-storage>/printers/_defaults.json` holds
  the fleet default printer + saved per-queue option presets; conflicting
  writes resolve last-write-wins by `written_at_ms` (Q5), matching the
  mesh-storage conflict model.

### 3.2 Job routing (the IPP layer)

The host peer shares its local queues over the overlay only:

* `cupsctl --share-printers` + per-queue `lpadmin -o printer-is-shared=true`.
* `cupsd` `Listen <overlay-ip>:631` — **bound to the Nebula overlay
  interface, never `0.0.0.0`** (open-mesh directive + §0.7 #10 public-port
  lint). A `<Location>` allowing the overlay CIDR (`10.42.0.0/16`).
* Imported queue on B is `ipp://<host-overlay>:631/printers/<queue>`, so
  a job on B travels the tunnel to A's `cupsd`, which filters + spools to
  the hardware. B needs no driver for the routing itself; the replicated
  PPD (Q4) is for capability/option presentation in B's print dialog and
  to keep legacy queues well-formed.

### 3.3 Worker (`mackesd::cups_sync`)

5 s tick (mirrors `gluster_worker` / `cups`-absent → silent no-op):

1. **Publish:** enumerate local queues (`lpstat`), write own
   `printers/<host>.json` + copy each legacy queue's PPD to the PPD store.
2. **Share:** ensure `cupsd` overlay Listen + `printer-is-shared` on each
   local queue.
3. **Import:** read the union of `printers/*.json` (minus self); for each
   remote queue, `lpadmin` an `<queue>@<host>` queue pointing at the
   host's overlay IPP endpoint (`-m everywhere`, or `-P <replicated ppd>`
   for legacy); prune `@host` queues whose host file disappeared.
4. **Defaults/presets:** reconcile `_defaults.json` LWW; apply the fleet
   default (`lpoptions -d`) + presets.
5. Auto-join on `EnrollmentCompleted`; pure-fn helpers
   (`own_record`, `import_plan`, `lpadmin_argv`, `prune_list`,
   `resolve_defaults_lww`) for unit testing without a live cupsd.

### 3.4 Profiles + packaging

`cups` + `cups-filters` hard-Required by base `mde-core`. The worker runs
on **headless** + **full** (a headless NAS can host *and* consume
printers); **lighthouse** skips it (routing-only, like it skips the
brick). Birthright enables `cups.service` + the overlay Listen on
headless/full.

## 4. Acceptance (bench-observable, ≥2-peer fleet)

1. Add a USB printer on peer A (`lpadmin` a local queue). Within the heal
   window, peer B's `lpstat -e` lists `<queue>@anvil`.
2. Print a test page from B to `<queue>@anvil`; the page comes out of A's
   physical printer (job routed over the overlay to A's cupsd).
3. `cupsd` on A is listening on the overlay IP only — `ss -ltn` shows
   `:631` bound to `10.42.0.x`, not `0.0.0.0`.
4. A legacy (non-IPP) printer's PPD appears under
   `mesh-storage/printers/ppd/anvil/` and B's imported queue presents its
   options.
5. Set the fleet default on A; within the window B's default
   (`lpstat -d`) follows; a conflicting set on B within the same window
   resolves LWW by timestamp.
6. Remove the printer on A; within the window B prunes `<queue>@anvil`.
7. A lighthouse peer runs no `cups_sync` worker (not in its worker
   roster) and has no `cups` service active.

## 5. Security review notes (DoD gate #8)

New surface: `cupsd` IPP on **TCP/631 bound to the Nebula overlay
interface only** (never a public bind — passes the §0.7 #10 lint). Reached
only by enrolled peers (Nebula is the auth/transport boundary; single
mesh passcode). Fits the open-mesh / flat-trust directive: every enrolled
peer may print to every shared printer, no per-printer ACL. No new D-Bus
method; control/observe is Bus + the worker. PPD files replicated through
`mesh-storage` are host-authored config, not executable.

## 6. Out-of-scope

* Per-printer / per-peer access control (flat-trust; Q3).
* Print-job accounting / quotas.
* Scan / fax (CUPS print only).
* Cross-arch driver packages (driverless + PPD only; exotic
  vendor-driver printers that aren't IPP-Everywhere and need a binary
  filter are unsupported — flagged as a risk).

## 7. Risks

* **Legacy binary-filter printers.** A non-IPP printer needing a
  vendor binary filter won't render correctly via a replicated PPD alone;
  driverless covers modern hardware, PPD covers PostScript/PCL-class
  legacy, but proprietary-filter printers are out of scope.
* **Host-asleep printing.** If the host peer is offline, its shared
  queues are unreachable (jobs queue on B until A returns). Expected for
  a physically-attached printer; the panel shows host reachability.
* **PPD staleness.** A driver update on A must re-publish the PPD; the
  5 s tick + `written_at_ms` handles it, but a hand-edited PPD on B is
  overwritten by the host's authority.
