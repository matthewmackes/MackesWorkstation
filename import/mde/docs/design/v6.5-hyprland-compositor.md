## 0. Why swap?

The visual-polish wins Hyprland delivers (compositor-level animations,
rounded corners, per-window decoration plugins, mark pills as native
window decoration) are the load-bearing reason. The motion-vocabulary
already targets a polished aesthetic at the app layer; Hyprland extends
the same vocabulary to the window-frame layer.

The migration also lets `mde-hypr-plugin` (a new C++ plugin) own the
three window-grain mechanics that Portal R12 needed compositor-native
hooks for: custom MDE layout, window rules, event-bridge to the Bus
broker for layout + rule events. Doing this against sway would have
meant either an out-of-tree compositor patch or accepting that those
mechanics live in Portal-side overlays that lag window motion.

**Mark state + mark-pill rendering live one process out** (re-locked
2026-05-27 via simplification-pass of the C++-codebase-size risk):
marks are held in `mackesd::workers::marks_state` (Rust, subscribed
to `hyprctl sockets2`) and pills are rendered by `mde-portal` as a
wlr-layer-shell overlay subscribed to mark deltas via Bus. Plugin
scope shrinks from 5 subsystems to 3; the cmake-via-build.rs bridge
in HYP-4 stops being load-bearing for the visible polish.

**Costs honestly accepted:** 33+ migration tasks across ~30 .rs files,
4 Cargo.toml updates, a new C++ build chain in-tree, an end-to-end
rewrite of Portal R12's sway-IPC integration points, and a v6.5 cut
that will not tag until operator HW bench is green.

## 2. Portal R12 redesign + WM mechanics (R2, Q11–Q20)

Portal R12 (locked 2026-05-26, Portal-41..Portal-59) defined 19 sway/i3
power-feature integrations. This section reframes each against
Hyprland equivalents. Worklist tasks under the HYP-* prefix supersede
the corresponding Portal-4N items.

| Q | Portal R12 item | Hyprland redesign |
|---|---|---|
| Q11 | Portal-41 auto-workspace-names | Named workspaces via `workspace = name:foo`; `workspace_namer` worker dispatches `renameworkspace <id> <name>` on first window event. |
| Q12 | Portal-42 tag-driven WS→output | Runtime `workspace_router` worker subscribes to workspace-create + dispatches `moveworkspacetomonitor`. |
| Q13 | Portal-43 tag autostart | `tag_autostart` worker subscribes to workspace-create + dispatches `exec` per tag manifest at `~/.config/mde/tags/<name>.toml`. |
| Q14 | Portal-44 tag layout | **Custom MDE layout** in `mde-hypr-plugin` implements Hyprland's `IHyprLayout` interface. |
| Q15 | Portal-45 Mod+r resize | **Mouse-grab dispatcher only**; keyboard submap retired. **Amends R12 Q44**. |
| Q16 | Portal-46 auto-marks by app_id | **Re-locked 2026-05-27:** `mackesd::workers::marks_state` subscribes to `hyprctl sockets2` (openwindow / closewindow / movewindow / activewindow), holds `HashMap<WindowAddr, Vec<String>>` in worker state, exposes `action/marks/{add,remove,list,match}` on Bus, publishes deltas on `event/marks/<addr>`. Hyprland has no native mark primitive, but the plugin doesn't need one — the state lives in Rust one process out. (Original lock: plugin owned the marks field; superseded.) |
| Q17 | Portal-47 mark pills | **Re-locked 2026-05-27:** `mde-portal` renders pills as a wlr-layer-shell `Top` surface (one per output) positioned by `hyprctl clients` geometry on focus + window-state-change events; subscribes to mark deltas on Bus topic `event/marks/<addr>`. Pill style read from design tokens. (Original lock: plugin rendered pills inside `renderWindow` callback; superseded by the simplification pass — Portal already owns shell chrome per v6.0 lock.) |
| Q18 | Portal-48 templates as Card | Portal generates `hyprctl --batch` dispatch sequence per template; one atomic IPC call per template apply. |
| Q19 | Portal-49 session restore | `session_persist` worker writes `~/.local/share/mde/session/<peer>.toml` every 60 s (GFS-replicated); replay seeds `windowrulev2` for re-launch position memory. |
| Q20 | Portal-50 window rules | Plugin owns window rules in C++; reads from plugin-specific config file at `~/.config/mde/window-rules.toml`. |

**`mde-hypr-plugin` consolidated scope** (delegates from Q9 + Q14 +
Q20; Q16 + Q17 retargeted out of plugin per the 2026-05-27
simplification re-lock):
- Custom MDE layout (`IHyprLayout` impl)
- Window rules engine
- Event bridge → Bus broker (publishes layout + rule-application
  events that aren't exposed via `hyprctl sockets2`)

This is a real C++ codebase but **substantially smaller post-
simplification** than the original 5-subsystem lock — marks live in
mded, pills in mde-portal layer-shell. The plugin retains the
mechanics that genuinely need compositor-internal hooks (layout +
rules); everything else moved one process out.

## 6. Risks

| Risk | Mitigation |
|---|---|
| **C++ plugin maintenance burden** | Plugin scope tightly bounded to **3 subsystems** post 2026-05-27 re-lock (layout / rules / event-bridge); marks + pill rendering moved to Rust mded + mde-portal. No further scope creep without explicit operator authorization. |
| **hyprland-rs API churn** | Pin to latest stable; bump per named cut; smoke-test the IPC surface in CI on every Hyprland-tag bump. |
| **Hyprland tag instability mid-development** | Tag pinning at named cuts only — `main` always builds against a known-good tag. |
| **C++ build chain in workspace** | Isolated to `crates/mde-hypr-plugin/`; build.rs handles the cmake invocation; rest of workspace untouched. |
| **Operator HW battery-life regression from compositor animations** | Durations capped at 120-150 ms windows / 100 ms ambient; blur scoped to Portal layer-shells only; tearing off; HW bench gate (§0.15) catches regression before cut. |
| **Portal R12 sway-IPC code becomes immediately dead** | Same-commit retire of sway-IPC; no co-existence; hyprland-rs ports happen in the same bundles as sway-IPC removals. |
| **Visual lock drift** (per-elevation shadows amend Classic ChromeOS) | Captured in §3 + governance §14 supersession table; CR-* tasks update accordingly. |

### 10.1 In-place re-lock 2026-05-27 (simplification pass)

After the 30-Q lock landed, an operator-driven risk-review pass
flagged the size of the C++ plugin as the single largest risk
to the v6.5 cut: HYP-12 + HYP-14/15/18 made `mde-hypr-plugin`
a substantial C++ codebase, with HYP-4's cmake-via-build.rs
bridge load-bearing for visible polish.

Operator picked simplification option 1 (move pill render +
mark state out of the plugin). The in-place re-lock:

- **Q16 retargets** marks state from C++ plugin to
  `mackesd::workers::marks_state` (Rust, subscribes to
  `hyprctl sockets2`, publishes via Bus).
- **Q17 retargets** pill rendering from C++ plugin to
  `mde-portal` as wlr-layer-shell overlay (subscribes to mark
  deltas from Bus).
- **Q9 + plugin consolidated scope** shrinks from 5 subsystems
  to 3 (layout + rules + event-bridge).
- **HYP-4 cmake bridge** stops being load-bearing for visible
  polish — failures degrade gracefully (layout / rules still
  matter, but Portal-side fallback covers most v3.x risk).
- **HYP-14 + HYP-15 worklist tasks** retarget to Rust crates
  per matching design updates (see `docs/PROJECT_WORKLIST.md`).

Per §0.14 authority hierarchy, the 2026-05-27 in-place re-lock
supersedes the original 2026-05-27 30-Q wording silently — no
separate "Conflicts" section. The original Q-table text above
has been updated in place; this metadata section records the
amendment trail.

Per §0.16, this is a scope **reduction** on a locked epic, not
an expansion — no fresh lock-lift needed; the original 2026-05-27
operator lift covers in-place narrowing.
