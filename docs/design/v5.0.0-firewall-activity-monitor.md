# v5.0.0 — Firewall activity monitoring (`firewall_monitor`)

**Status:** Locked 2026-05-29 via 7-Q survey (round 2 of a 15-Q
print + firewall-monitor session).
**Target release:** v5.0.0 — **operator-elevated to cut-blocking core**
(§11.1) 2026-05-29, overriding the §0.17 minimal-core default.
**Depends on:** firewalld (already present; `firewall_preset` worker
owns the port/zone preset) + `mesh-storage` mount + Bus + the MON/alert
notification path.
**Worklist:** `docs/PROJECT_WORKLIST.md` § "FWMON-1..FWMON-7".

## 1. Headline

Every peer watches its firewall for denied inbound attempts ("who's
knocking on closed ports"), publishes them to the mesh, and raises a
notification when one source crosses a port-scan / brute-force threshold.
A new `mackesd::firewall_monitor` worker reads firewalld's `LogDenied`
journal entries, filters out the mesh's own + established traffic, and
appends net-new external denials to a per-host JSONL on `mesh-storage`
that any peer can union for a fleet-wide, centerless security view —
surfaced as an Activity section in the existing Workbench firewall panel.

## 3. Architecture

### 3.1 Pipeline

```
firewalld:  firewall-cmd --set-log-denied=all   (FWMON-1, idempotent)
            └─ denied packets → kernel log → journald
               (prefix "filter_*_DROP: ... SRC=.. DST=.. DPT=.. PROTO=..")

mackesd::firewall_monitor (5s tick, every peer):
  read journal since last cursor (sd-journal / `journalctl -o json --cursor`)
   └─ parse each DROP line → DeniedEvent {ts, src_ip, dport, proto, iface}
   └─ FILTER (Q14): drop if src/dport is overlay (UDP/4242, TCP/443 LH)
                    or RELATED,ESTABLISHED; keep net-new external only
   └─ append kept events → <mesh-storage>/firewall/<host>.jsonl
   └─ trim lines older than 7 days (Q15)
   └─ threshold check (Q13): per-source-IP count in the last <window>;
      crossing N → publish event/firewall/<host> on the Bus
                   → FDO notification via the MON/alert relay

Workbench firewall panel "Activity" (Q12):
  read union of <mesh-storage>/firewall/*.jsonl → recent table +
  top-source rollup + per-peer counts
```

### 3.2 Records

`<mesh-storage>/firewall/<host>.jsonl` — one JSON object per line:
`{ts_ms, host, src_ip, dport, proto, iface}`. Append-only within the tick,
rolling-trimmed to 7 days. Own-file authority (PEERVER pattern); readers
union all `firewall/*.jsonl` for the fleet view.

### 3.3 Worker (`mackesd::firewall_monitor`)

* Separate worker from `firewall_preset` (Q15) — preset = config
  mutation, monitor = observability; single-responsibility per the
  worker convention.
* 5 s tick; persists a journal cursor so each tick reads only new
  entries. Silent no-op when `LogDenied` is off / no journal access.
* Pure-fn helpers for testing without a live journal:
  `parse_denied_line`, `is_overlay_or_established` (the Q14 filter),
  `threshold_tripped(events, src, window, n)`, `trim_older_than(lines, cutoff)`.
* `RestartPolicy::Always`; `ShutdownToken` select.

### 3.4 Alerting

Threshold default: ≥ N (e.g. 10) denials from a single `src_ip` within a
rolling window (e.g. 60 s) → one Bus `event/firewall/<host>` (deduped per
source per window so a sweep fires once, not per-packet) → FDO desktop
notification via the existing `mde-alert-emit` / `alert_relay` path
(MON-3/MON-4). Observe-only fallback if the alert path is down (records
still land).

### 3.5 Profiles + packaging

Every profile runs firewalld, so `firewall_monitor` runs on **all three
profiles** (lighthouse especially — a public-IP VPS sees the most
external knocking). `FWMON-1` flips `LogDenied` in birthright on every
profile.

## 4. Acceptance (bench-observable, ≥2-peer fleet)

1. `firewall-cmd --get-log-denied` returns `all` after install on every
   profile.
2. From an off-mesh host, knock on a closed port of peer A
   (`nc -z A <closed-port>`). Within a tick a line appears in
   `mesh-storage/firewall/anvil.jsonl` with the right `src_ip` + `dport`.
3. Peer B's Workbench firewall **Activity** section shows A's denial in
   the per-peer rollup (read from the union).
4. Overlay traffic (UDP/4242 between peers) and established connections
   produce **no** records (Q14 filter).
5. A rapid port-scan (≥ N hits from one source in the window) raises
   exactly one FDO notification, not one per packet.
6. Records older than 7 days are trimmed from the JSONL on the next tick.
7. `firewall_monitor` appears in every peer's worker roster (all
   profiles), distinct from `firewall_preset`.

## 5. Security review notes (DoD gate #8)

No new listening port — the worker only *reads* journald + appends to
`mesh-storage`. The Bus `event/firewall/<host>` topic is mesh-internal
(Nebula-bounded). `LogDenied=all` increases journald volume; the 7-day
trim + the Q14 filter bound it. Source IPs of external scanners are
recorded to a mesh-replicated file readable by every enrolled peer —
acceptable under flat-trust (the data is "who attacked us," shared
fleet-wide on purpose). No `{{exec}}` template, no D-Bus method.

## 6. Out-of-scope

* Accepted-connection / full-traffic accounting (Q10 — Netdata's job).
* Automated blocking / fail2ban-style banning (monitoring only; an
  auto-ban responder could be a later epic).
* Per-source geo/IP reputation enrichment.
* Configurable thresholds UI (ship sensible defaults; tuning is a
  follow-up).

## 7. Risks

* **Journal format drift.** `LogDenied` line format is stable on Fedora
  firewalld but parser must tolerate field reordering — parse by `KEY=`
  tokens, not position.
* **High-scan volume.** A sustained scan could grow the JSONL fast within
  the 7-day window; per-source dedup on the alert + the trim bound it,
  but a pathological case could pressure the GFS quota (shared with the
  mesh-storage quota guard).
* **Cursor loss.** If the persisted journal cursor is lost, the worker
  re-reads from "now" (not the whole journal) to avoid a backfill storm —
  a small gap is preferred to a flood.
