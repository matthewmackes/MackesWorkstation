# Mackes Bus — Node Notification Distribution platform

**Locked:** 2026-05-25 via 104-Q poll across 26 rounds (30 candidate
features brainstormed; 24 survived; 7 cut with rationale below).

**Status:** Design-locked, not yet built. Awaiting user "execute" /
"iterate" / "ship the Bus" per CLAUDE.md §0.5 before implementation.

**Worklist:** `docs/PROJECT_WORKLIST.md` § "v6.x BUS-1..7 — Mackes Bus".
**Memory:** `project_v6_x_mackes_bus.md`.

---

## 0. Axiom

Mackes Bus is the **single bus** for every event the mesh produces or
consumes — notifications, alerts, clipboard sync, FDO desktop
notifications, webhook ingress, mesh-internal pub/sub. It is built on
self-hosted **ntfy** brokers running on every peer over the **Nebula**
overlay, with persistence on the **GlusterFS mesh-home**.

It replaces, in a single v6.x cut:

- v5.1 GF-17 mesh notification bus (hard cut, delete + rewrite callers)
- v2.6 MON Netdata alert routing (parallel-write window; legacy
  `~/.local/share/mde/alerts/` JSONL retained for external consumers)
- Standalone FDO `org.freedesktop.Notifications` handling (republished
  to `fdo/<app>` topics)
- Any per-app clipboard sync (becomes the `clipboard/sync` topic +
  custom `mde-clipd` daemon)

It folds into the v6.0 Portal epic.

## 2. Broker architecture (Round 2)

| Decision | Lock |
|---|---|
| Broker topology | **Every peer runs its own broker, gossip-synced** |
| Persistence | **SQLite on GlusterFS mesh-home** (single shared DB) |
| Discovery | **mDNS on the Nebula overlay** (`_mackes-bus._tcp`) |
| Failover budget | **< 30s best-effort** (every peer is a broker; failover is mostly moot) |

**Reconciliation with Round 8 file-tree:** SQLite-on-GFS is the
**queryable index**; the per-topic file tree
(`~/.local/share/mde/bus/<topic-path>/<ulid>.json`) is the
**authoritative store** and inotify watch surface. Mirrors the MON
alert pattern.

## 3. Topic model (Rounds 3-4)

| Decision | Lock |
|---|---|
| Topic naming | **Slash hierarchy** (`fleet/sec`, `peer/laptop/alerts`, `mon/cpu`) |
| Default topics | **~12 curated** covering MON, GF-17 use cases, system events, mesh changes |
| Creation auth | **Self-serve** (any peer with the mesh passcode publishes; topic exists) |

| Per-priority retention overrides | **urgent=forever, high=30d, default=7d, min=24h** |
| Lifecycle | **Full rename + alias + delete** via Workbench panel |

## 7. Webhook ingress (Round 7)

| Decision | Lock |
|---|---|
| Adapter | **ntfy's built-in webhook publisher** (no custom adapter) |
| Authentication | **Nebula source-IP only** (overlay membership = auth) |
| Transform | **YAML rules file** per peer at `~/.config/mde/bus-hooks.yaml` |
| Built-in adapters | **6**: generic JSON + GitHub + Gitea + Sonarr/Radarr + UPS/NUT + Home Assistant |

**Architecture note:** "ntfy's built-in publisher" + "YAML transform rules"
means a thin Rust shim consumes inbound webhooks, evaluates the YAML
rules (match path / headers / body, extract fields), and calls ntfy's
publish API. The shim lives inside mded.

## 11. Global Clipboard (Rounds 11, 25-26 — feature #31)

Folded into Mackes Bus rather than shipped as a standalone subsystem.
Mackes Bus carries the messages; `mde-clipd` is the local Wayland
daemon.

| Decision | Lock |
|---|---|
| Sync scope | **All-peer auto-sync** (every copy on any peer becomes every peer's clipboard) |
| History depth | **Unlimited** (governed by `clipboard/sync` topic retention) |
| Encryption | **None** — Nebula transport is enough; flat-trust mesh |
| Payload size | **≤ 64 KB inline; larger → GFS reference + URL** |
| Wayland daemon | **Custom `mde-clipd`** using `wlr-data-control-unstable-v1` (Rust) |
| MIME types | **All** sync (text, images, application/* blobs) |
| Per-app exclusion | **None** — use Super+Shift+C modifier to skip a copy |
| Conflict resolution | **Merge text with separator** on simultaneous copies |
| Skip-sync modifier | **Super+Shift+C** (mde-clipd intercepts at Wayland layer) |
| Super+V layout | **Centered floating popover** — scrollable card list, type-to-filter, j/k navigation, enter pastes |
| Pin / favorite | **Via Mackes universal tag system** (clipboard entries are taggable) |
| Phone round-trip | **Bidirectional always** via KDC2 — phone↔mesh clipboard |
| CLI surface | **None** — clipboard is hotkey-only (Super+V) |

KDC2 bridge + Breadcrumb wire-up — Phase 1 ships a full demo.

## 24. Open architectural tensions (resolved at write-time)

1. **SQLite (Round 2) vs file-tree (Round 8) persistence.** Resolved
   as: file-tree authoritative (matches MON pattern, inotify-friendly),
   SQLite as queryable index for the Workbench audit/history view.
2. **`min` priority (Round 5: silent) vs Breadcrumb TTL (Round 19:
   30s fade).** Resolved as: `min` produces no Breadcrumb segment at
   all. The Round 19 TTL applies to `default`/`high`/`urgent` only.
3. **"ntfy built-in publisher" (Round 7) vs "YAML transform rules"
   (Round 7).** Resolved as: thin Rust shim in mded consumes inbound
   webhooks, evaluates YAML, calls ntfy publish API. Not literally
   "raw ntfy publisher with no shim."
4. **Feature #6 phone reach (KDC2 only) vs Round 12 lock (both KDC2 +
   ntfy app dedup).** Round 12 supersedes the brainstorm wording.
